Welcome to my Rust learning experience

I've heard that learning your first programming language takes roughly a year, learning your second language takes a few months, learning your third programming language will take a few days. I may have the time frames a bit wrong, but the idea is that learning your first language means that you have to be introduced to all the base computing concepts. This can take quite a bit of time to really drill into habit. After that, learning the syntax of your first programming language may feel like writing with your off-hand for a while, but it eventually starts to click. Then, when it comes to learning your second language, you only have to learn a slightly different syntax, and how the programming language approaches the common computing principles.

That being said, while I have mainly been working in javascript, I have hopped around and played with some other languages, such as Python, R, C, C++, and PHP. I never really pursued any of them, and I never built any substantial projects with them. Most of them, such as C and C++ I only ever got as far as some basic syntax and printing some lines to the console. That means, that so far, I have only really ever programmed in type agnostic languages with automatic garbage collection. Even with my limited knowledge, I can imagine how important these two tools are to developing performant software. Now, one might think that since I am somewhat experienced in javascript, the natural next step to a statically typed language would be Typescript. I know it is inevitable, but I have a severe aversion to Typescript. Something about the overhead of having to build the linter into each project, and then learning to implement types and all the types associated with each package I may bring into to any package, just seems exhausting and fruitless. Beyond all that, I'd have to constantly fight the urge to use the any type everytime a struggle to get something to work. Finally, it still doesn't solve the lack of control of garbage collection, so I am still leaning on some sort of cruch in my development.

So all this leads me to Rust. Rust is of course statically typed and has its own flavor manual garbage collection. Rust has a strong community and excellent documentation. So, let's get started.

The Set-up

With javascript, you can open any text editor and start chugging out code and see its effects in the browser. Unfortunely Rust will not work this way. We need to set-up development environment.

Installing Rust

For this set-up I will be working on a linux system and I'll be using Neovim as my text editor.

In order to write in Rust, we must install Rust and Rust package manager. From what I understand there are a couple of options for this, but I personally simply installed rustup which handles Rust versions as well as installs the rust package manager called cargo. I am on an arch based distro and I use yay as my system package manager, so I can simply run the following: 

#home/
$ yay -S rustup

Great, now we can create a directory for our project and create our first program. 

#home/
$ cargo new helloworld

The above command creates a directory within your current directory and initiates a new project including a git repository. Within this project directory, at /src/main.rs we will find the following code: 

#helloworld/src/main.rs

fn main() {
  println!("Hello, World!");

So we already have a first program. Starting from our home directory, we can run the follwing commands see our first program in action. 

#home/
$ cd helloworld
$ cargo run
